在开发java企业级应用时，我们可能需要导入各种第三方的jar包，比如spring的jar包、dao框架mybatis的jar包，日志系统slf4j的jar包及其实现logback或log4j的jar包等等，jar包的版本以及版本之间的冲突都会让我们无比麻烦，这个时候我们就需要使用构建工具来帮助我们管理我们的项目结构 。
Maven是用java写的一个强大的构建工具，多用于java项目的构建。实际上，Maven的开发者宣称：Maven不仅仅只是一个构建工具，但是现在，我们可以仅仅把他当做一个构建的工具。当你学会使用Maven之后，你就会明白Maven究竟是什么了。

什么是构建工具

大家可能会疑惑，什么叫做构建工具。构建工具是自动化构建软件项目的工具，一般来说，构建一个软件项目包括以下的过程

  • 生成源代码（如果项目中使用了自动生成代码）
  • 从源代码生成文档
  • 编译源代码
  • 将编译后的代码打包成jar或zip文件
  • 安装代码包到服务器、仓库或其他地方。
    以上这些操作是软件项目经常需要执行的操作，通过自动化完成这些过程，可以提高速度并减少错误。
    除了Maven外常见的构建工具还有Ant和Gradle

安装Maven

安装Maven，首先需要去官网下载Maven压缩包
1.安装Maven之前需要确保已经安装Java，并设置好JAVA_HOME环境变量(Maven 3.0.5需要Java 1.5及以上，而Maven3.3.3需要用Java 7以上)
2.解压Maven压缩包
3.设置M2_HOME指向你解压Maven的目录地址
4.设置M2环境变量指向(%M2_HOME%\bin)
5.将M2添加到Path变量中(;%M2%)
6.打开命令行工具，执行命令mvn -version,如果能输出maven的版本号则安装成功。

使用Maven创建第一个项目

Maven常用命令

mvn -v查看maven的版本
mvn compile编译源代码，生成class字节码文件
mvn test 运行测试代码
mvn package 打包项目，生成安装包
mvn clean 删除target文件夹(target目录下主要是生成的字节码文件和测试报告)
mvn install 安装jar包到本地仓库

使用Archetype自动建立项目的骨架

archetype(原型)，可以帮我们创建项目的骨架（即项目的文件结构），它就像是项目结构的一套模板。
常用的maven artichetype plugin有:
org.apache.maven.archetypes:maven-archetype-quickstart
org.apache.maven.archetypes:maven-archetype-site
org.apache.maven.archetypes:maven-archetype-webapp
你也可以自定义archetype，这在一个具有统一规范的公司十分有用，用这个archetype创建的项目的文件结构都是相同的。
